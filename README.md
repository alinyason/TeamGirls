1. Инициализация игры
Игрок запускает игру через GameManager.

Spawner создает уровень, запрашивая у PoolManager:

Монстров (инициализирует их сложность)

Клубнику (настраивает связанные обучающие элементы).

Уровень считается готовым (Level Ready).
2. Основной игровой цикл
Движение и коллизии
Игрок управляет персонажем (бег/прыжки).

Проверяется столкновение с блоками (декоративными объектами).

Сбор клубники
При столкновении с клубникой:

Активируется обучающий элемент (задача/вопрос).

Если игрок решает задачу:

Клубника добавляется в инвентарь (Add Strawberry).

Объект возвращается в пул (ReturnToPool).

При ошибке — клубника остается.

Бой с монстрами
Атака прыжком:

Если у игрока ≥3 клубник → монстр умирает за 1 удар (TakeDamage(2)).

Если <3 клубник → требуется 2 удара (TakeDamage(1)).

После победы:

Монстр возвращается в пул.

Счетчик убийств обновляется (Add Kill)
3. Завершение игры
GameManager проверяет условия:

Стандартная победа: пройти уровень.

Секретное видео: собрать 9 клубник и убить 7 монстров.

1. Основные системы
GameManager

Центральный управляющий компонент

Контролирует процесс спавна объектов через Spawner

Отслеживает игровой прогресс (сбор 9 клубник, убийство 7 монстров)

Определяет условия победы и активацию специального финала

Spawner

Отвечает за создание игровых объектов

Запрашивает объекты у PoolManager для оптимизации производительности

Инициализирует врагов и клубники на уровне

PoolManager

Реализует шаблон Object Pool для управления памятью

Содержит пулы переиспользуемых объектов (враги, клубники)

Обеспечивает быстрый доступ к префабам

2. Игровые объекты
Player

Основной управляемый персонаж

Механики:

Атака прыжком (1-2 удара в зависимости от клубник)

Сбор клубник с решением задач

Взаимодействие с окружением

Strawberry

Коллекционные предметы

Содержат обучающие элементы

При сборе:

Активируют головоломку

Восстанавливают здоровье

Дают игровые очки

LearningElement

Обучающие головоломки

Проверяют правильность ответов

Могут предоставлять подсказки

Enemy

Вражеские персонажи

Логика получения урона

Разное поведение при 1/2 ударах

Block

Элементы окружения

Обеспечивают платформинг

Не содержат игровой логики
3. Ключевые взаимодействия
Механика сбора:

Игрок сталкивается с клубникой

Активируется LearningElement

При успешном решении - клубника добавляется в инвентарь

Система боя:

При 3+ клубниках: 1 удар для убийства

Менее 3 клубник: требуется 2 удара

Управление памятью:

Уничтоженные враги и собранные клубники возвращаются в PoolManager
