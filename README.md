1. Инициализация игры
Игрок запускает игру через GameManager.

Spawner создает уровень, запрашивая у PoolManager:

Монстров (инициализирует их сложность)

Клубнику (настраивает связанные обучающие элементы).

Уровень считается готовым (Level Ready).
2. Основной игровой цикл
Движение и коллизии
Игрок управляет персонажем (бег/прыжки).

Проверяется столкновение с блоками (декоративными объектами).

Сбор клубники
При столкновении с клубникой:

Активируется обучающий элемент (задача/вопрос).

Если игрок решает задачу:

Клубника добавляется в инвентарь (Add Strawberry).

Объект возвращается в пул (ReturnToPool).

При ошибке — клубника остается.

Бой с монстрами
Атака прыжком:

Если у игрока ≥3 клубник → монстр умирает за 1 удар (TakeDamage(2)).

Если <3 клубник → требуется 2 удара (TakeDamage(1)).

После победы:

Монстр возвращается в пул.

Счетчик убийств обновляется (Add Kill)
3. Завершение игры
GameManager проверяет условия:

Стандартная победа: пройти уровень.

Секретное видео: собрать 9 клубник и убить 7 монстров.

1. Основные системы
GameManager

Центральный управляющий компонент

Контролирует процесс спавна объектов через Spawner

Отслеживает игровой прогресс (сбор 9 клубник, убийство 7 монстров)

Определяет условия победы и активацию специального финала

Spawner

Отвечает за создание игровых объектов

Запрашивает объекты у PoolManager для оптимизации производительности

Инициализирует врагов и клубники на уровне

PoolManager

Реализует шаблон Object Pool для управления памятью

Содержит пулы переиспользуемых объектов (враги, клубники)

Обеспечивает быстрый доступ к префабам

2. Игровые объекты
Player

Основной управляемый персонаж

Механики:

Атака прыжком (1-2 удара в зависимости от клубник)

Сбор клубник с решением задач

Взаимодействие с окружением

Strawberry

Коллекционные предметы

Содержат обучающие элементы

При сборе:

Активируют головоломку

Восстанавливают здоровье

Дают игровые очки

LearningElement

Обучающие головоломки

Проверяют правильность ответов

Могут предоставлять подсказки

Enemy

Вражеские персонажи

Логика получения урона

Разное поведение при 1/2 ударах

Block

Элементы окружения

Обеспечивают платформинг

Не содержат игровой логики
3. Ключевые взаимодействия
Механика сбора:

Игрок сталкивается с клубникой

Активируется LearningElement

При успешном решении - клубника добавляется в инвентарь

Система боя:

При 3+ клубниках: 1 удар для убийства

Менее 3 клубник: требуется 2 удара

Управление памятью:

Уничтоженные враги и собранные клубники возвращаются в PoolManager

1. Центральные компоненты системы
GameManager (Ядро игры)
Главный координатор, отвечающий за:

Состояние игры (GameState)

Сохранение и загрузку прогресса

Проверку условий победы

Координацию работы всех подсистем

LevelSystem (Управление уровнем)
Включает:

LevelLoader - загрузка и управление уровнем

CameraController - камера, следующая за игроком

ParallaxManager - эффекты параллакса для фона
ObjectSpawner - генерация всех объектов уровня (блоки, враги, клубники)

2. Системы взаимодействия с игроком
PlayerSystem (Персонаж)
Полностью описывает игрока:

PlayerController - обработка ввода (движение, прыжки)

PlayerPhysics - физика персонажа

PlayerAnimation - анимации

PlayerCombat - боевая система (удары по врагам)

UISystem (Интерфейс)
Отвечает за отображение информации:

HUDController - игровой HUD (здоровье, очки)

MenuManager - система меню

DialogSystem - диалоги и подсказки

3. Игровые механики
LearningSystem (Обучающая система)
Реализует сбор клубники:

StrawberryManager - логика сбора

ProblemGenerator - генератор задач

ProblemTypes - типы вопросов (математика, головоломки)

EnemySystem (Враги)
Управление противниками:

EnemySpawner - размещение врагов

EnemyAI - поведение врагов

EnemyCombat - система боя с врагами

4. Ключевые взаимосвязи
GameManager → Все системы
Центральный координатор управляет всеми подсистемами

PlayerSystem ↔ EnemySystem
Взаимодействие игрока с врагами (атаки, получение урона)

PlayerSystem ↔ LearningSystem
Сбор клубники и решение задач

LevelSystem → (EnemySystem, LearningSystem)
Размещение врагов и клубник на уровне

GameManager ↔ UISystem
Обновление интерфейса на основе состояния игры

5. Особенности архитектуры
Четкое разделение ответственностей
Каждая система отвечает только за свою область

Гибкость и расширяемость
Легко добавить:

Новые типы врагов

Дополнительные виды задач

Дополнительные элементы интерфейса

Оптимизированные зависимости
Минимизация связей между системами

Поддержка основных механик игры:

Сбор клубники с решением задач

Бой с врагами (1-2 прыжка)

Условия победы (прохождение уровня + дополнительные цели)

6. Преимущества данной структуры
Удобство разработки
Разные программисты могут работать над разными системами параллельно

Простота тестирования
Каждую систему можно тестировать независимо

Поддержка модификаций
Легко вносить изменения в конкретные механики, не затрагивая другие системы

Читаемость
Понятная структура для новых членов команды

Состав игровых объектов:

Центральный контейнер GameWorld содержит:

Текущий уровень (Level_1)

Экземпляр игрока (PlayerCharacter)

Менеджер игры (MainGameManager)

Систему интерфейса (GameUI)

Детализация уровня:

Уровень включает:

2 типа блоков: обычный (ground) и платформа (platform)

2 врага: Гумба (1 HP) и Купа (2 HP)

2 клубники (1 собрана, 1 доступна)

Фон с параллакс-эффектом

Состояние игрока:

Позиция: (120, 300)

Здоровье: 3 единицы

Собрано: 2 клубники

Убито: 3 врага

Текущее действие: бег ("running")

Визуальные элементы:

Интерфейс отображает:

Счётчик клубник (2)

Счётчик убитых врагов (3)

Отсутствие активных окон

Ключевые взаимосвязи:

Жёсткие связи (композиция):

GameWorld → Level_1

GameWorld → PlayerCharacter

Гибкие связи (агрегация):

Level_1 → Block/Enemy/Strawberry

Взаимодействия:

Игрок ↔ Враги (атаки)

Игрок ↔ Клубники (сбор)

Игрок ↔ Блоки (платформы)

Стилистические особенности:

Цветовая дифференциация классов:

GameWorld (голубой)

Level_1 (зелёный)

PlayerCharacter (розовый)

Чёткое разделение экземпляров объектов

Диаграмма служит для:

Визуализации текущего игрового состояния

Анализа расположения объектов

Понимания взаимосвязей между компонентами

Отладки игровых механик

Для других игровых моментов можно модифицировать:

Количество и положение объектов

Состояние персонажа

Параметры взаимодействий

1. Основные системы персонажа
Движение (Movement)
Прыжки (jumps) - базовый механизм перемещения, позволяющий преодолевать препятствия и атаковать врагов сверху

Таймер уровня (Level timer)
Отслеживает время прохождения уровня (дублирование "time" вероятно означает разные временные метрики)

Может использоваться для ограничения времени на уровень или бонусов за скорость

2. Боевая система (Combat system)
Получение очков (getting points) - начисление баллов за:

Уничтожение врагов

Сбор предметов

Быстрое прохождение

3. Взаимодействие с миром
Игровая система (Game system)
Коллизии с окружением (collisions with environment) - обработка:

Столкновений с платформами

Взаимодействия с блоками

Границ уровня

Особые взаимодействия (Interaction)
Фермерство (trains farming) - возможно:

Выращивание растений

Сбор урожая

Уход за NPC

4. Прогресс и жизненная система
Жизни и очки (Lives and points)
Рестарт при Game Over - система:

Возврата к контрольным точкам

Потери прогресса

Управления жизнями

Прогресс (Progress)
Выполнение задач (progress done) - отслеживание:

Собранных предметов

Убитых врагов

Открытых зон

Следующий уровень (next level) - условия перехода:

Выполнение квестов

Достижение конца уровня

Набор нужного количества очков

5. Особые способности
Разрушение блоков (Block crush)
Уничтожение врагов - уникальный механик:

Возможность разрушать блоки

Использование обломков как оружия
